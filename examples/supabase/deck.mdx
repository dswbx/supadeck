## What it <em style={{ color: "#ff6b6b" }}>isn't</em>
<Divider />
<CardGrid columns={3}>
   <Card title="Not a replacement" tone="red">
      Not a drop-in replacement for Supabase. Production apps belong on
      Supabase.
   </Card>
   <Card title="Not full parity" tone="red">
      100% compatibility is not the goal, only the most common features for AI
      builders.
   </Card>
   <Card title="High-traffic apps" tone="red">
      Not aiming for high-traffic or large-scale apps. Use Supabase for that.
   </Card>
</CardGrid>

---

<Tag>What it is</Tag>
## Supabase <em>supplement</em>
<Divider />
<CardGrid columns={3}>
   <Card title="For AI builders">
      <p>
         Built for local and prototype workflows in tools like Lovable, Bolt,
         and similar AI-first products.
      </p>
   </Card>
   <Card title="~60% feature parity">
      <p>
         Enough of the common surface area to build and iterate quickly without
         chasing edge-case parity.
      </p>
   </Card>
   <Card title="Seamless upgrade">
      <p>
         When you are ready for production, moving to Supabase should require no
         code changes.
      </p>
   </Card>
</CardGrid>

## vs <em>Supabase</em>

<Divider />

| Metric | supalite | Supabase (t4g.nano) | | ------ | -------- | ------------------- |
| Idle cost | **Storage only** — $0.20/gb _~0.5mb per small project_ | **~$0.71** compute before pausing _$0.0042/hr × 168h (7 days)_ | | Scale to zero | **Yes** — after 10s inactivity | Pauses after ~7 days | | Cold start | **\<1s** | up to minutes |
